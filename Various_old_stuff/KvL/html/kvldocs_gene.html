Search Option: Gene

This is not actually a search option, you can only input a single gene name here. Databases with gene names that are well curated are available for a few species only (e.g. Human and mouse). Fansnip incorporates the gene database from NCBI, which in turn uses curated databases if available (e.g. Hugo for human gene names). For species without curated databases NCBI will assign gene names based on homologue predictions.

Fansnip will only retrieve a gene name if it exists in the NCBI database. Alias databases have not been implemented as dealing with ambiguities often can only be done by user intervention. The rationale behind Fansnip is to minimize user intervention. There is one exception: if the gene name itself is not present for a non-human species, but it is the same as a human gene name, the orthologue of that gene will be retrieve on the premise that gene names will only be identical if there is some kind of homology relationship. This may not always be the case.

As Fansnip uses primarily Ensembl gene id's no gene can be retrieved if there is not a proper cross database (NCBI-EnsEMBL) annotation. Unfortunately this is the case for a sizeable portion of the genes.


What to expect:

Let's assume you have typed in a valid gene name, say 'IGF1'.

It may take some time for results to appear. Then you will see:

chicken gene IGF1 (insulin-like growth factor 1 (somatomedin C)) has NCBI gene id 418090 and EnsEMBL gene id ENSGALG00000012755

You will also notice a button stating 'submit'. If you press this button, the gene(s) that were found can be process in the SNPsearch phase.


If you did not submit a valid name you will be notified about it and you can choose to start a new query by pressing 'new'.

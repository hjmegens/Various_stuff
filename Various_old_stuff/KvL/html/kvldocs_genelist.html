Search Option: Genelist

This is not actually a search option, you can only input gene name or EnsEMBL ids here, or a combination of the two. The file should be plain text, with one gene name or id per line and no additional information or whitespace. The text file can be uploaded from your computer by pressing 'browse' and selecting the file and then hitting 'submit'.

Note that EnsEMBL gene ids are species specific, which means that if you did not select 'chicken' in this example, no genes would be retrieved for those two ids.


Databases with gene names that are well curated are available for a few species only (e.g. Human and mouse). Fansnip incorporates the gene database from NCBI, which in turn uses curated databases if available (e.g. Hugo for human gene names). For species without curated databases NCBI will assign gene names based on homologue predictions.

Fansnip will only retrieve a gene name if it exists in the NCBI database. Alias databases have not been implemented as dealing with ambiguities often can only be done by user intervention. The rationale behind Fansnip is to minimize user intervention. There is one exception: if the gene name itself is not present for a non-human species, but it is the same as a human gene name, the orthologue of that gene will be retrieve on the premise that gene names will only be identical if there is some kind of homology relationship. This may not always be the case.

As Fansnip uses primarily Ensembl gene id's no gene can be retrieved if there is not a proper cross database (NCBI-EnsEMBL) annotation. Unfortunately this is the case for a sizeable portion of the genes. You can circumvent this problem by using EnsEMBL ids in stead.
